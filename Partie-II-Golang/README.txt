How to run the program ?

From your command line, execute 

> go run KnapsackBruteForce.go p1.txt


p1.txt est le fichier contenant les données utilisées pour l'entrée du programme.
La sortie du programme p1.sol est créé et ajouter directement dans le meme folder dans lequel ces fichiers
sont exécutés. 

Peu importe le nom du fichier utilisé l'entrée : pour (filename.txt), la solution est (filename.sol) à condition que ce fichier 
contienne les memes données que p1.txt;


*** Le temps d'execution optimal est affichée par le programme, le seuil correspondant
 ainsi que le nombre de goroutines créé ***

Autres exigences: 

- Une explication concernant l'approche utilisé est contenue dans le fichier explanation_approach.pdf (ce fichier contient egalement un graphe)
- Le fichier 'autre_fichier.txt' (disponible pour la Partie I en OOP) contient lui aussi des donnees pour le 
  Knapsack et peut etre run comme suit:
  
  > go run KnapsackBruteForce.go autre_fichier.txt

  Et sa solution est stockée dans le fichier 'autre_fichier.sol' et prend un plus de temps d'exécution en raison de la 
  quantité de ses données.
